# REST
- a design pattern (set of rules) to provide a standard way for clients and servers to communicate
### Statelessness
- It's when the server gets a request, fulfills it if possible, then forgets everything about the request and client who made the request (sessions, state etc.)
### URL Params
- Variable inside the URL that acts as a placeholder for the real value
### Query Strings
- Allow us to filter the result we get back, they are included in URLs
- Syntax : `URL/endpoint?query=<value>`
- you can have more queries by adding `&` followed by the query
